# 无人机智能巡检系统产品需求文档

## 1. 背景

无人机智能巡检系统面向高速公路巡检场景，围绕大疆机场 3、视觉算法、数据标注、模型训练和告警工单形成闭环能力。系统由江西省交投数智科技有限公司建设、运营和运维，服务江西省内多个高速公路运营管理分中心。

一期建设聚焦“平台主链路 + 14 个视觉算法上线验收”，不追求一次性覆盖所有后续算法和所有外部系统形态。

## 2. 产品目标

- 建立可租赁、可审计、可扩展的无人机巡检平台。
- 支持从飞行任务创建、审批、下发、遥测、媒体同步，到抽帧、推理、告警、工单、复核的主链路闭环。
- 支持数据标注、数据集封版、训练、评估、模型发布和回滚闭环。
- 支持平台方全功能运营，客户租户按授权使用飞控、标注、分析结果和告警工单。
- 接入一期 14 个真实视觉算法，并满足离线指标、回放、人工抽检和双硬件适配要求。

## 3. 用户与角色

| 用户类型 | 典型角色 | 核心诉求 |
| --- | --- | --- |
| 平台方 | 平台管理员、平台运维员、算法工程师、标注管理员 | 统一管理租户、设备、算法、模型、权限、审计和运维 |
| 客户租户 | 飞控管理员、飞控操作员、分析查看员、标注管理员、审计查看员 | 使用本租户授权设备执行巡检，查看分析结果、告警、工单和标注任务 |
| 实施与运维 | 实施工程师、运维工程师 | 完成客户环境部署、外部系统适配、日志监控、故障定位 |
| 测试与验收 | 测试工程师、客户验收人员 | 验证平台闭环、算法指标、租户隔离、网络边界和性能指标 |

## 4. 一期范围

### 4.1 必建范围

| 范围 | 内容 |
| --- | --- |
| 基础平台 | 登录认证、租户、RBAC、功能授权、数据授权、审计、配置、对象存储、消息队列、异步任务中心、日志监控 |
| 飞控平台 | 大疆机场 3 适配网关、模拟器、设备/机巢台账、航线、任务、审批、下发、遥测、异常、媒体同步 |
| 视觉分析平台 | 媒体入库、断点续传、抽帧、分析任务、推理调用、结果聚合、告警、工单、人工复核、误报漏报回流 |
| 数据标注平台 | 样本池、标注任务、标注工具、初检/复检、数据集导出、样本来源追踪 |
| 训练与模型管理 | 数据集版本、训练任务、模型评估、模型版本、灰度发布、回滚、双硬件一致性验证 |
| 算法接入 | 一期 14 个真实视觉算法，统一输出结构、阈值策略和验收报告 |
| 系统验收 | 性能、安全、弱网、断点续传、租户隔离、功能授权、数据授权、网络边界、双硬件一致性 |

### 4.2 不纳入一期

- 自研底层飞控、安全返航算法或无人机固件。
- 客户租户默认训练和模型发布能力。
- 每个运营管理分中心独立物理部署。
- 大规模分布式训练和自动超参搜索。
- 二三期算法真实上线验收。
- 复杂跨摄像头跟踪和全量实时视频智能编排。

## 5. 核心业务流程

### 5.1 飞控巡检流程

1. 平台方或客户租户在授权范围内维护设备、机巢、GIS 资产和航线。
2. 用户创建飞行任务，选择航线、设备、计划时间、巡检目标和媒体策略。
3. 任务进入审批，支持内置审批流，并预留 OA/空域审批适配。
4. 审批通过后，通过大疆机场 3 适配网关下发任务。
5. 平台接收 DJI 设备侧回执、遥测、异常和媒体回传。
6. 媒体完成入库、校验和抽帧后，可提交分析任务。

### 5.2 分析与处置流程

1. 分析任务按媒体、算法和优先级创建。
2. 系统通过统一推理服务调用 NVIDIA 或国产后端。
3. 原始检测结果聚合为业务事件。
4. 低置信、定位不确定或高风险事件进入人工复核。
5. 确认后的事件生成告警和工单。
6. 误报、漏报、人工修正进入反馈样本池。

### 5.3 标注与训练流程

1. 样本从抽帧、人工反馈或外部导入进入样本池。
2. 标注任务按算法、标签和样本范围分发。
3. 标注结果经过初检、复检和质量统计。
4. 数据集封版后进入训练或评估。
5. 模型评估达标后生成版本和双硬件制品。
6. 模型按租户和算法进行灰度发布，异常时支持回滚。

## 6. 功能需求

### 6.1 基础平台

- 支持本地账号，并预留 LDAP/CAS/OAuth2/OIDC 统一身份适配。
- 支持 `PLATFORM_OPERATOR` 和 `CUSTOMER_TENANT` 两类租户。
- 支持 RBAC、`user_tenant_roles`、`tenant_feature_entitlements`、`tenant_data_grants`。
- 支持写操作、审批、发布、删除、越权、跨租户切换审计。
- 支持对象存储、消息队列和异步任务中心。
- 所有写接口支持 `Idempotency-Key`。

### 6.2 飞控平台

- 支持大疆机场 3 设备绑定、状态同步、航线上传、任务下发、任务控制、遥测订阅、媒体同步和异常事件。
- 支持设备、机巢、航线、任务、审批、遥测和飞行日志管理。
- 支持任务状态机，不允许非法状态跳转。
- 支持 WebSocket 推送 1 秒级遥测。
- 支持弱网、断点续传、重复回调和 checksum 校验。

### 6.3 视觉分析平台

- 支持视频、图片、抽帧图片统一入库。
- 支持按时间间隔、关键帧或任务策略抽帧。
- 支持按媒体、算法、优先级创建异步分析任务。
- 支持统一推理服务调用，不直接绑定具体硬件后端。
- 支持检测结果聚合、去重、事件定位、证据链、告警和工单。
- 支持人工复核、误报漏报回流和样本候选池。

### 6.4 数据标注平台

- 支持样本池、标注任务、任务分配和样本来源追踪。
- 支持 bbox、polygon、polyline、scene_label、density 等标注类型。
- 支持初检、复检、退回、修正和质检统计。
- 支持数据集导出；客户租户默认关闭导出，需单独授权。
- 数据集 `FROZEN` 后不可修改。

### 6.5 训练与模型管理

- 支持数据集版本、训练任务、评估、模型版本、后端制品、发布和回滚。
- 支持 NVIDIA 与国产后端 `runtime_artifacts`。
- 支持客户数据训练授权校验，未授权返回 `DATA_GRANT_412`。
- 未达标模型不得发布。
- 同模型双后端输出差异超过阈值时阻断发布。

## 8. 非功能需求

| 指标 | 通过标准 |
| --- | --- |
| 实时分析延迟 | 1080p、25fps、单路实时流、3 个一期算法，P95 <= 3 秒，失败率 < 1% |
| NVIDIA 离线吞吐 | 1920x1080 图片，batch 启用，每卡 >= 1000 帧/分钟，或按锁定 GPU 型号给出更高基线 |
| 国产卡离线吞吐 | 国产卡型号、驱动、SDK、镜像锁定后形成性能基线；未锁定前不得进入 U7 硬件验收 |
| 并发飞行监控 | 20 架次同时在线，1 秒级遥测，Web 端 10 并发用户，P95 响应 < 2 秒 |
| 告警推送 | 检测结果到工单/告警创建 P95 <= 10 秒，不含人工复核等待 |
| 数据可靠性 | 弱网、断网、恢复、重复回调、重复分片场景下不丢文件，checksum 一致，不重复入库 |
| 安全审计 | 核心写操作审计完整可查，保留策略 >= 180 天 |
| 租户隔离 | 未授权访问返回 `TENANT_403` 或 `TENANT_404`，不能读取或推断其他租户数据 |
| 网络边界 | 仅大疆机场 3 适配网关可访问 DJI Cloud API，其他平台无法直接访问互联网 |
| 硬件一致性 | 同模型 NVIDIA 与国产后端输出差异不超过发布阈值 |

## 9. 验收准入

- 平台闭环：飞行任务创建到媒体入库、抽帧、分析、告警、工单、复核可跑通。
- 算法闭环：样本、标注、数据集、训练、评估、发布、回滚可跑通。
- 客户租户：可使用飞控、标注、分析结果，不能默认使用训练和发布。
- 平台方：可全功能管理，跨租户操作有审计。
- 网络：内网服务不能绕过大疆机场 3 适配网关访问互联网。
- 硬件：真实硬件未锁定不得宣称硬件验收通过。
- 算法：M2 必须使用真实模型，模拟算法不得计入通过。

## 10. 主要风险

| 风险 | 等级 | 应对 |
| --- | --- | --- |
| 平台方与客户租户角色混淆 | 高 | 强制 `tenant_type`，初始化脚本和 U7 验收核对 |
| 客户租户误获得训练/发布权限 | 高 | 默认关闭，`FEATURE_403` 用例验证 |
| 客户数据训练授权不清 | 高 | 使用 `tenant_data_grants`，训练前强制校验 |
| DJI API 权限或证书不可用 | 高 | 先建设适配层模拟器，真实接入独立验收 |
| 网络边界不清 | 高 | 仅适配网关白名单出站，U1/U7 做出口阻断测试 |
| 国产卡型号未定 | 高 | 未锁定前只允许模拟后端契约测试 |
| 样本不足或偏差 | 高 | 优先真实场景采样，公开数据仅用于预训练 |
