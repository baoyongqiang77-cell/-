## 1. 开发结论

本系统一期建设目标是完成平台主链路和14个视觉算法上线验收。系统采用“平台方全功能 + 客户租户授权功能”的多租户模式。江西省交投数智科技有限公司作为平台方，使用飞控、视觉分析、数据标注、训练管理、模型发布、运维审计等全部功能；高速公路运营管理分中心作为客户租户，默认使用飞控、数据标注、算法分析结果、告警/工单等授权功能，算法训练和模型发布默认不开放。

一期飞控能力以大疆机场3（DJI Dock 3）和 DJI Cloud API 集成为边界。系统不自研底层飞控安全逻辑，只负责任务编排、航线管理、审批、机巢控制适配、遥测监控、异常联动和媒体同步。网络上仅“大疆机场3适配网关”可访问互联网/DJI Cloud API，其余平台默认部署在政务云内网。

### 2.2 一期不做或不承诺范围

| 不纳入范围 | 说明 |
| --- | --- |
| 自研底层飞控 | 真实飞行控制、底层安全策略、设备侧执行结果以 DJI 设备和 Cloud API 回执为准 |
| 客户租户默认训练能力 | 客户租户默认不开放训练和模型发布；如需开放训练功能，必须配置 `tenant_feature_entitlements` 并通过专项测试验收；如需使用客户租户数据训练、评估或跨租户统计，还必须具备有效 `tenant_data_grants` |
| 每租户物理隔离 | 一期采用逻辑多租户，不承诺每个运营管理分中心独立部署数据库、对象存储和算力 |

| 二三期算法真实上线 | 二三期算法仅保留主数据、标签schema和模拟占位，不进入一期真实模型验收 |

## 5. 飞控平台功能

### 5.1 业务边界

飞控平台负责业务编排，不直接承担底层飞行控制。所有设备动作通过"大疆机场3适配网关"调用 DJI Cloud API 或模拟器；业务平台不得散落调用 DJI 接口。U1-F01 编码与模拟器实现以架构定版 3.5.1 节"关键回调payload示例"为字段基准，模拟器至少需覆盖该节列出的绑定成功、遥测推送、低电量异常、媒体上传完成四类样例及其失败分支。

### 5.2 功能清单

| 编号 | 功能 | 开发要求 | 验收口径 |
| --- | --- | --- | --- |
| U1-F01 | 大疆机场3适配网关 | 封装设备绑定、状态同步、任务下发、遥测订阅、媒体同步、异常处理 | 模拟器和真实 API 调用使用同一业务接口 |
| U1-F02 | 设备/机巢台账 | 管理大疆机场3、无人机、载荷、边缘节点、固件、在线状态 | 设备归属 tenant_id，客户租户只能看本租户设备 |
| U1-F03 | GIS资产台账 | 管理道路、桥梁、边坡资产，支持最小资产台账 | 航线、任务、事件可绑定资产 |

任务状态必须与架构定版 4.1 保持一致，不得另行简化。状态跳转必须校验当前状态、目标状态、操作权限、设备回执和审计要求。

| DRAFT | 草稿可编辑，不触发设备侧动作 |

| DISPATCHING | 正在调用 DJI 适配层 |
| DISPATCHED | 设备侧已接受任务 |
| EXECUTING | 飞行执行中 |
| PAUSED | 人工暂停或设备侧暂停 |
| RETURNING | 返航或人工接管中 |
| LOST_LINK | 通信中断，平台等待设备侧回执 |
| COMPLETED | 飞行完成，可有媒体回传 |
| ABORTED | 异常终止，仍可能存在部分媒体 |
| MEDIA_SYNCING | 媒体同步中 |
| PARTIAL_MEDIA_READY | 部分媒体可分析 |
| MEDIA_READY | 全量媒体可分析 |
| ANALYSIS_READY | 可提交分析任务 |
| CANCELLED | 未执行前取消 |

### 6.1 事件定位与GIS映射

检测结果必须先保留图像像素坐标和帧时间，再结合无人机遥测、航线、相机姿态、GIS资产几何进行空间映射。一期自动定位最低验收到 L1；具备客户 GIS 数据、坐标系转换和桩号映射时验收到 L2。无法可靠定位到资产时，事件进入 `NEEDS_GEO_REVIEW`，由人工修正后再生成正式工单。

| 等级 | 名称 | 所需输入 | 输出 | 一期要求 |
| --- | --- | --- | --- | --- |
| L0 | 图像证据级 | 图片/视频帧、frame_time | 证据图、像素坐标 | 算法调试和人工复核，不作为正式工单定位 |
| L1 | 资产级 | 航线、无人机粗略位置、资产几何 | road/bridge/slope asset_id | 一期最低生产验收目标 |
| L2 | 桩号/区段级 | L1 + 坐标系转换 + 路线桩号映射 | asset_id + stake_no/segment_id | 一期优先目标；GIS数据不足时进入人工复核 |
| L3 | 构件/精确空间级 | L2 + 相机标定 + 姿态 + 时间同步 | 桥梁构件/设施部件/误差半径 | 二三期目标，一期不承诺自动达到 |

## 8. 训练与模型管理功能

### 8.1 权限边界

训练和模型发布默认仅平台方可用。客户租户如需使用训练能力，必须通过 tenant_feature_entitlements 单独开通；如训练、评估或跨租户统计涉及客户租户数据，还必须通过 tenant_data_grants 明确数据用途、范围、有效期和授权状态。

### 8.2 功能清单

| 编号 | 功能 | 开发要求 | 验收口径 |
| --- | --- | --- | --- |
| U5-F01 | 数据集版本 | 管理 DRAFT、FROZEN、ARCHIVED 等数据集状态 | 数据集包含 source_tenant_scope 和 checksum |
| U5-F02 | 数据授权校验 | 训练、评估或跨租户统计前校验客户数据是否授权；客户租户自行使用本租户数据做标注不等同于训练授权 | 未授权数据返回 DATA_GRANT_412 |

## 14. 验收准入

| 验收项 | 准入条件 |
| --- | --- |
| 架构一致性 | 功能实现不得突破架构定版中飞控、网络、多租户、硬件适配边界 |
| 平台闭环 | 飞行任务创建到媒体入库、抽帧、分析、告警、工单、复核可跑通 |
| 算法闭环 | 样本、标注、数据集、训练、评估、发布、回滚可跑通 |
| 客户租户 | 客户租户可使用飞控、标注、分析结果，不能默认使用训练和发布 |
| 数据集导出 | 客户租户未授权导出时返回 FEATURE_403；授权后只能导出本租户样本 |
| 平台方 | 平台方可全功能管理，并且跨租户操作有审计 |
| 网络 | 内网服务不能绕过大疆机场3适配网关访问互联网 |
| 硬件 | 同一模型在 NVIDIA 与国产后端接口一致，真实硬件未锁定不得宣称硬件验收通过 |
