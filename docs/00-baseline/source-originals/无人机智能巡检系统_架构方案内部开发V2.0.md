- 租赁服务对象：江西省内多个高速公路运营管理分中心
- 定版日期：2026-06-16
- 定版来源：基于内部开发定版 V1.6 最终审查通过后固化
- 文档定位：内部开发团队架构基线，作为后续软件功能开发文档与测试文档的上游依据

## 1. 定版结论与一期范围

本方案将原 V1.0 概念架构调整为内部开发架构基线。后续软件功能开发文档、单元拆分、测试文档和验收脚本均以本文件定义的边界、状态机、接口约定、数据模型和验收指标为准。

一期分为两个明确里程碑：

| 里程碑 | 目标 | 算法要求 | 可验收输出 |
| --- | --- | --- | --- |
| M1 平台闭环可用 | 飞控、媒体入库、抽帧、模拟推理、告警、工单、标注、训练、发布流程跑通 | 可使用模拟算法服务，不要求真实模型指标达标 | U0-U5 单元测试文档、平台闭环演示记录、模拟算法回放结果 |
| M2 一期最终验收 | 14个一期真实算法接入并通过指标、回放、人工抽检和双硬件适配要求 | 必须使用真实模型；模拟后端不能替代真实算法验收 | 一期14算法验收测试文档、模型评估报告、UAT记录、系统级验收报告 |

- 无人机飞控平台采用大疆机场3（DJI Dock 3）与 DJI Cloud API 集成边界，平台负责业务编排与监管，不自研底层飞控安全逻辑。
- 训练与推理平台采用统一模型注册、统一推理接口、双硬件运行时适配，覆盖 NVIDIA GPU 与国产 AI 加速卡。
- 外部系统能力均通过适配器接入。适配器必须区分开发模拟、试运行兜底、生产必接三种模式，不能把模拟器误作为生产能力。
- 多租户采用“平台方租户 + 客户租户”模式。江西省交投数智科技有限公司是平台建设、运营和运维单位，具备全功能与受审计的跨租户运维能力；高速公路运营管理分中心作为客户租户，默认使用飞控、标注、算法分析结果、告警工单等授权功能，算法训练与模型发布默认由平台方集中管理。
- 网络采用分区部署模式：大疆机场3适配网关位于互联网出口/DMZ区，其余飞控业务服务、分析、标注、训练、数据存储、AI算力和运维审计服务默认部署在政务云内网。

## 3. 总体架构

- 四个平台边界：飞控平台、视觉算法分析平台、数据标注平台、算法训练与管理平台。
- 飞控平台只做任务编排、航线管理、审批、机巢控制指令编排、遥测监控和异常联动，真实飞行控制由 DJI 设备能力承担。
- 分析平台采用媒体入库、抽帧、异步推理、聚合去重、告警工单的任务链路。
- 标注平台负责样本导入、任务分发、初检/复检、数据集封版。
- 训练平台负责训练、评估、模型制品、双硬件适配、灰度发布和回滚。

### 3.1 平台边界

| 平台 | 一期职责 | 不承担范围 |
| --- | --- | --- |
| 飞控平台 | 设备管理、航线/任务、审批、机巢指令编排、遥测监控、飞行日志 | 底层飞控、安全返航算法、无人机固件 |
| 视觉分析平台 | 视频/图片入库、抽帧、任务调度、推理服务调用、结果聚合、告警工单、人工复核回流 | 复杂跨摄像头跟踪和全量实时视频智能编排 |
| 数据标注平台 | 图像导入、标注任务、初检/复检、数据集导出、样本统计 | 大规模外包结算、众包平台对接 |
| 训练管理平台 | 训练任务、评估对比、模型版本、上架、回滚、灰度、双后端一致性验证 | 自动超参搜索和大规模分布式训练 |
| 客户现有平台 | 统一身份、OA审批、GIS路网、对象存储/政务云资源、既有工单平台按接口集成 | 客户接口不可用时需提供适配器或临时内置替代能力 |

### 3.2 多租户隔离设计

一期支持逻辑多租户，租户模型必须区分平台方与客户租户。江西省交投数智科技有限公司是平台建设、运营和运维单位，拥有平台方租户和全功能后台；江西省内多个高速公路运营管理分中心是被服务客户租户，按租赁授权使用飞控、数据标注、算法分析结果等功能。多租户不是可选功能，所有业务数据必须具备租户归属；但一期不承诺为每个运营管理分中心提供物理隔离部署。

| 隔离层 | 一期设计 | 开发约束 | 验收方式 |
| --- | --- | --- | --- |
| 租户模型 | `tenants` 表作为租户根实体，支持 `PLATFORM_OPERATOR` 与 `CUSTOMER_TENANT` 两类租户 | 初始化江西省交投数智科技有限公司平台方租户，并按运营管理分中心初始化客户租户；所有业务表必须有 `tenant_id` 或能通过父级实体追溯到 `tenant_id` | 任选任务、媒体、事件、模型、工单，均可追溯租户和租户类型 |
| 用户与权限 | 用户来自统一身份或本地账号镜像；角色授权需绑定租户范围和功能范围 | `users`、`roles`、`user_tenant_roles`、`tenant_feature_entitlements` 共同决定访问范围 | 平台方可按权限运维客户租户；A客户租户用户不能查询、修改B客户租户数据 |
| 功能授权 | 平台方默认拥有飞控、分析、标注、训练、模型发布、运维审计等全功能；客户租户默认开放飞控、标注、分析结果、告警/工单 | 训练、模型发布、跨租户数据汇聚等高权限功能默认仅平台方可用，客户租户需单独授权 | 客户租户访问未授权训练/发布接口时应被拒绝并写审计 |
| 设备与航线 | 设备、机巢、航线、巡检资产归属租户 | 大疆机场3绑定必须写入 `tenant_id`，跨租户复用设备必须显式授权 | 未授权租户不能看到设备、航线和遥测 |
| 媒体与对象存储 | 对象路径按 `tenant_id/project_id/mission_id` 分层 | 上传、抽帧、模型制品路径不得省略租户前缀 | 对象存储路径和数据库记录租户一致 |
| 消息队列 | 消息体必须包含 `tenant_id`；队列可按环境选择共享队列或租户队列 | 消费者必须做租户校验，不能只信任消息来源 | 构造跨租户消息时应被拒绝或隔离 |
| 算法与模型 | 一期算法主数据可平台共享；模型训练和模型发布由平台方集中管理；阈值策略、发布记录可租户私有 | `algorithm_releases` 和 `algorithm_thresholds` 必须支持租户维度覆盖；客户租户默认只能查看授权模型和分析结果 | 同一算法在不同客户租户可使用不同阈值和模型版本，客户租户不能越权发布模型 |
| 数据训练授权 | 客户租户数据默认归属本租户；如用于平台方统一训练，需记录数据授权范围 | 通过 `tenant_data_grants` 记录数据用途、范围、有效期和状态 | 未授权客户数据不得进入训练数据集 |
| 审计日志 | 审计日志必须记录 `tenant_id`、actor、resource_type、resource_id | 跨租户访问失败也要写审计 | 审计查询可按租户过滤 |

功能授权基线：

| 功能域 | 平台方租户 | 客户租户默认权限 | 说明 |
| --- | --- | --- | --- |
| 飞控平台 | 全量管理、跨租户运维、设备接入配置 | 管理本租户授权设备、航线、任务、审批和遥测 | 客户租户只能看到本中心相关设备与任务 |
| 视觉分析平台 | 全量配置、任务调度、结果复核、跨租户统计 | 查看本租户分析任务、检测结果、告警和工单 | 客户租户不直接管理底层推理运行时 |
| 数据标注平台 | 全量标注配置、质检、数据集汇聚 | 创建/处理本租户标注任务，查看本租户样本 | 是否参与跨租户训练数据集由数据授权决定 |
| 训练管理平台 | 训练、评估、模型版本、发布、回滚、双硬件一致性验证 | 默认无训练和发布权限；可按授权查看模型评估摘要 | 客户租户较少使用训练功能，一期不作为默认租赁能力 |
| 运维审计 | 平台运维、租户配置、日志审计、告警监控 | 查看本租户审计与操作记录 | 平台方跨租户操作必须有审计留痕 |

租户隔离级别：

| 级别 | 说明 | 一期是否支持 |
| --- | --- | --- |
| L1 逻辑隔离 | 同一套应用、数据库和对象存储，通过 `tenant_id`、租户类型、功能授权、权限和路径隔离 | 支持，作为一期默认 |

### 3.3 网络分区与访问控制设计

网络设计原则：飞控业务需要对接 DJI Cloud API，但不代表整个飞控平台都通互联网。系统只允许“大疆机场3适配网关”访问互联网/DJI Cloud API，其余服务默认运行在政务云内网，通过内网 API、消息队列和对象存储交换数据。

| 网络区 | 部署组件 | 网络权限 | 关键控制 |
| --- | --- | --- | --- |
| 互联网出口/DMZ区 | 大疆机场3适配网关、证书/回调入口、必要的反向代理 | 仅允许访问 DJI Cloud API 白名单域名/IP；仅开放必要回调端口 | 出站白名单、TLS证书、API签名、最小端口、访问日志 |
| 内网业务区 | 飞控业务服务、任务审批、Web后台、分析平台、标注平台、训练管理平台 | 不直接访问互联网；只访问DMZ适配网关、数据区、AI服务区 | 服务间认证、RBAC、审计、API网关 |
| 数据区 | PostgreSQL/PostGIS、对象存储、消息队列、模型仓库、配置中心 | 仅允许内网业务区、AI服务区按最小权限访问 | 数据库账号分权、bucket路径隔离、队列ACL、备份 |
| AI服务区 | 推理服务、训练服务、GPU/国产AI加速卡节点 | 不访问互联网；访问对象存储、模型仓库、消息队列 | 镜像白名单、模型制品校验、资源隔离 |
| 运维审计区 | 日志、监控、审计、告警、堡垒机 | 只接收日志/指标，运维访问经堡垒机 | 操作留痕、最小权限、告警策略 |
| 客户现有系统区 | 统一身份、OA/空域审批、GIS、工单平台 | 通过内网专线/API对接 | 接口签名、IP白名单、失败重试 |

访问控制矩阵：

| 来源 | 目标 | 协议/方式 | 是否允许 | 说明 |
| --- | --- | --- | --- | --- |
| 大疆机场3适配网关 | DJI Cloud API | HTTPS/MQTT/WebSocket，具体以DJI Cloud API版本为准 | 允许 | 必须走白名单、证书、签名和审计 |
| DJI Cloud API | 大疆机场3适配网关 | HTTPS回调/消息通道，具体以DJI Cloud API版本为准 | 条件允许 | 只允许到DMZ网关，不直达内网业务服务 |
| 内网飞控业务服务 | 大疆机场3适配网关 | 内网REST/gRPC/消息队列 | 允许 | 业务服务通过适配网关间接控制设备 |
| 分析/标注/训练平台 | 互联网 | 直接访问 | 禁止 | 依赖包、模型、镜像需离线导入或走受控制品库 |
| AI服务区 | 对象存储/模型仓库 | 内网S3/API | 允许 | 只读/读写权限按任务授予 |
| Web用户 | Web后台/API网关 | 内网或客户VPN | 允许 | 不直接暴露公网 |

### 3.6 双硬件推理运行时

| 抽象层 | NVIDIA 路线 | 国产 AI 加速卡路线 | 统一约束 |
| --- | --- | --- | --- |
| 模型格式 | PyTorch -> ONNX -> TensorRT Engine | PyTorch/ONNX -> 厂商模型格式或厂商ONNX兼容运行时 | 模型注册中心保留原始模型、ONNX、后端制品与校验集结果 |
| 推理服务 | TensorRT/ONNX Runtime CUDA Provider | 国产后端适配器封装厂商 SDK、运行时或 ONNX 兼容执行器 | 对分析平台只暴露统一 gRPC Predict 接口 |
| 候选生态 | NVIDIA T4/A10/L4 等，以客户GPU资源为准 | 昇腾、寒武纪、昆仑、海光等作为待确认候选，不在架构里默认绑定某一家 | 国产卡型号未锁定前，只能做接口契约和模拟后端验收 |
| 运行环境 | x86_64/ARM64 Linux，NVIDIA驱动、CUDA、TensorRT版本需入库 | 麒麟OS/ARM64或客户指定OS，厂商驱动、SDK、容器镜像需入库 | 每个模型版本必须记录运行时镜像、驱动、设备型号、后端制品 digest |
| 一致性验证 | 输出框、类别、置信度与后处理结果入库 | 输出字段保持一致，可允许少量数值差异 | 同一测试集结果差异超过阈值必须阻断发布 |

### 3.6.1 统一推理 gRPC 接口草案

本草案是 U3 推理运行时编码的起点，NVIDIA 与国产后端适配器必须实现同一接口，禁止各自定义返回结构。字段命名与 5.3 节 `schema:detection_geometry`、`schema:runtime_config` 保持一致；接口契约测试以本草案为基准。

```protobuf
syntax = "proto3";
package inference.v1;

service InferenceService {
  rpc Predict (PredictRequest) returns (PredictResponse);
  rpc HealthCheck (HealthRequest) returns (HealthResponse);
}

message PredictRequest {
  string inference_job_id = 1;
  string algorithm_id = 2;
  string model_version_id = 3;
  string frame_asset_id = 4;
  bytes image_data = 5;          // image_data 与 image_uri 二选一
  string image_uri = 6;
  int32 image_width = 7;
  int32 image_height = 8;
  string backend = 9;            // nvidia | domestic
  map<string, string> runtime_params = 10; // 覆盖 runtime_config 默认值，如 batch_size
}

message PredictResponse {
  string inference_job_id = 1;
  string status = 2;             // SUCCEEDED | FAILED | TIMEOUT
  string error_code = 3;         // 复用 6.2 错误码，如 INFER_504
  repeated Detection detections = 4;
  int32 latency_ms = 5;
  string backend = 6;
  string model_version_id = 7;
}

message Detection {
  string label = 1;
  double confidence = 2;
  oneof geometry {
    BBox bbox = 3;
    Mask mask = 4;
    Polyline polyline = 5;
  }
  map<string, string> attributes = 6; // helmet_missing、severity 等算法特有字段
}

message BBox { double x = 1; double y = 2; double width = 3; double height = 4; }
message Mask { string encoding = 1; bytes data = 2; }   // encoding: rle | polygon_json
message Polyline { repeated Point points = 1; }
message Point { double x = 1; double y = 2; }
```

字段与数据模型映射：

| protobuf 字段 | 对应数据模型字段 | 说明 |
| --- | --- | --- |
| PredictResponse.detections | detection_results.label / confidence / geometry_json | 落库前按 `schema:detection_geometry` 转换；bbox/mask/polyline 三选一对应该 schema 的 type 字段 |
| PredictResponse.latency_ms | inference_jobs.latency_ms | 用于 U3 一致性报告与 8 章性能基线 |
| PredictRequest.backend / PredictResponse.backend | inference_jobs.backend、runtime_artifacts.backend | 必须与发起调用时选定的后端一致，不允许后端静默切换 |
| PredictResponse.error_code | 6.2 节错误码（如 INFER_504、RUNTIME_428） | 失败时必须复用统一错误码，不得自造错误字符串 |
| PredictRequest.runtime_params | runtime_artifacts.runtime_config_json（`schema:runtime_config`） | 仅允许覆盖已登记的运行时配置项 |

3.6 节"一致性验证"以本接口的 `PredictResponse` 为比对对象：同一 `frame_asset_id`、同一 `model_version_id`，NVIDIA 与国产后端返回的 `detections` 数量、`label`、bbox/mask 几何重合度（IoU）、`confidence` 差异需落入发布阈值内，超过阈值按 8 章"硬件一致性"指标阻断发布。

| bridge_assets | id, bridge_code, bridge_name, route_code, stake_no, geom, structure_parts | 桥梁资产台账，结构部件可用于桥梁病害定位 |
| slope_assets | id, slope_code, route_code, start_stake, end_stake, side, geom | 边坡资产台账，支持左右幅和桩号范围 |
| asset_segments | id, asset_id, segment_type, geom, stake_start, stake_end | 细分路段、桥跨、边坡区段 |
| coordinate_transforms | id, source_crs, target_crs, method, params_json, version | 坐标系转换记录，至少支持 WGS84、GCJ-02、CGCS2000 之间的配置化转换 |
| event_asset_links | id, event_id, asset_id, asset_type, stake_no, offset_m, confidence | 检测事件与道路/桥梁/边坡资产的绑定结果 |

### 3.8 GIS定位精度分级

| 等级 | 名称 | 所需输入 | 输出 | 一期用途 |
| --- | --- | --- | --- | --- |
| L0 | 图像证据级 | 图片/视频帧、frame_time | 仅给出证据图和像素坐标 | 算法调试和人工复核，不作为正式工单定位 |
| L1 | 资产级 | 航线、无人机粗略位置、资产几何 | road/bridge/slope asset_id | 一期最低生产验收目标 |
| L2 | 桩号/区段级 | L1 + 坐标系转换 + 路线桩号映射 | asset_id + stake_no/segment_id | 一期优先目标；GIS数据不足时进入人工复核 |
| L3 | 构件/精确空间级 | L2 + 相机标定 + 姿态 + 时间同步 | 桥梁构件/设施部件/误差半径 | 二三期目标，一期不承诺自动达到 |

定位规则：检测结果先保留图像像素坐标和帧时间，再结合无人机遥测、航线、相机姿态、GIS资产几何进行空间映射。一期自动定位验收到 L1，具备客户GIS数据和桩号映射时验收到 L2。无法可靠定位到资产时，事件进入 `NEEDS_GEO_REVIEW`，由人工修正后再生成正式工单。

### 5.2 租户字段与继承规则

| 规则 | 开发约束 | 验收方式 |
| --- | --- | --- |
| 租户上下文来源 | 业务请求的租户上下文来自登录令牌、统一身份映射或受控的 `X-Tenant-Id`；普通客户端不得在请求体里自行指定 `tenant_id` | 伪造请求体 `tenant_id` 时应被拒绝并写入审计 |
| 表字段要求 | 租户根业务表必须显式包含 `tenant_id`；子表可通过父表继承租户，但查询和写入时必须校验父子租户一致 | 数据库迁移审查 + 跨租户用例测试 |
| 租户类型要求 | 平台方租户可拥有跨租户运维角色；客户租户只能访问本租户资源和已授权功能 | 平台方账号可切换服务租户，客户账号不可切换 |
| 功能授权要求 | 访问模块前必须校验 `tenant_feature_entitlements`；训练、模型发布、跨租户统计默认仅平台方可用 | 客户租户访问未授权训练/发布接口返回 FEATURE_403 |
| 数据授权要求 | 客户租户数据进入平台方训练集前必须存在有效 `tenant_data_grants` | 未授权样本创建数据集或训练任务应失败并写审计 |
| 对象路径要求 | 媒体、抽帧、数据集、模型制品路径必须包含 `tenant_id` 或租户专属 bucket/schema 映射 | 抽样检查 storage_uri、artifact_uri |
| 查询隔离要求 | 列表、详情、导出、WebSocket订阅、异步任务查询均必须自动附加租户过滤 | A租户账号不能读取B租户资源 |

| 审计要求 | 成功、失败、越权、跨租户拒绝、管理员切换租户均必须记录 `tenant_id`、actor、resource_type、resource_id | 审计样例核查 |

## 6. API 与错误码约定

- 外部业务接口采用 REST + JSON；实时遥测和任务进度采用 WebSocket；内部推理服务采用 gRPC。
- 所有写接口必须支持 Idempotency-Key；重复请求返回同一业务结果，不重复创建任务。
- 所有异步任务返回 task_id，并提供状态查询、取消、重试和失败原因字段。
- 所有敏感动作记录 audit_log，包括操作者、来源IP、请求体摘要、前后状态、结果。
- 所有业务接口必须从 Token、统一身份映射或受控 `X-Tenant-Id` 解析租户上下文；普通用户不得通过请求体指定或覆盖 `tenant_id`。
- 所有列表、详情、导出、WebSocket订阅和异步任务查询必须自动附加租户过滤；跨租户访问返回租户级错误码并写审计。
- 所有模块入口必须校验 `tenant_feature_entitlements`；客户租户默认不能访问训练、模型发布、跨租户统计和平台运维接口。
- 客户租户数据进入平台方训练、评估或跨租户统计前，必须校验有效 `tenant_data_grants`。
- 业务 API 默认部署在内网业务区，不直接访问互联网；调用 DJI Cloud API 的能力只能由大疆机场3适配网关提供。

### 6.2 错误码

| 错误码 | 含义 | 处理策略 |
| --- | --- | --- |
| AUTH_401 | 未认证或 Token 过期 | 重新登录或刷新 Token |
| PERM_403 | 权限不足 | 记录审计，提示申请权限 |
| TENANT_403 | 当前用户无目标租户访问权或试图伪造 `tenant_id` | 拒绝请求，记录跨租户审计 |
| TENANT_404 | 资源不存在于当前租户上下文 | 返回不可见，避免泄露其他租户资源存在性 |
| FEATURE_403 | 当前租户未开通目标功能 | 拒绝访问，提示联系平台方开通或调整授权 |
| DATA_GRANT_412 | 数据未获得训练、评估或跨租户统计授权 | 阻断数据集封版、训练或统计任务，返回缺失授权范围 |
| IDEMP_409 | 幂等键冲突 | 返回原请求摘要并拒绝不一致请求 |
| STATE_409 | 当前状态不允许目标操作 | 返回当前状态、允许动作、最后变更人 |
| MISSION_422 | 任务参数不满足飞行或审批条件 | 返回字段级校验错误 |
| DJI_502 | DJI 适配层调用失败 | 可重试错误进入补偿队列，不可重试进入人工处理 |
| GIS_422 | 坐标或资产匹配失败 | 进入人工定位复核 |
| MEDIA_415 | 媒体格式不支持 | 拒绝入库并保留失败原因 |
| MEDIA_499 | 分片上传未完成或 checksum 不一致 | 触发续传或重新上传 |
| INFER_504 | 推理超时 | 按算法配置重试，超过阈值标记失败 |
| MODEL_412 | 模型未满足发布准入 | 阻断发布，返回缺失指标 |
| RUNTIME_428 | 指定硬件后端未完成验证 | 阻断切换或发布 |

### 6.3 统一错误响应体结构

所有错误码必须包装为统一 JSON 结构，禁止各模块自定义错误字段名；成功响应保持业务字段直出，不强制包装，仅错误响应使用本结构。

```json
{
  "code": "TENANT_403",
  "message": "当前用户无目标租户访问权",
  "request_id": "req_8f3a1c2d",
  "details": {
    "resource_type": "mission",
    "resource_id": "m_1024",
    "field_errors": []
  }
}
```

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| code | 是 | 取值见 6.2 节错误码表，禁止返回未在表中登记的错误码 |
| message | 是 | 面向用户的简要说明，不得包含系统内部敏感信息（如SQL、堆栈、内部路径） |
| request_id | 是 | 链路追踪ID，与 audit_logs 关联，便于排障和审计核对 |
| details | 否 | 结构化补充信息：字段级校验错误（MISSION_422 的 field_errors）、缺失指标（MODEL_412）、缺失授权范围（DATA_GRANT_412 的 missing_scope）等按错误码扩展 |

U0-F10 异步任务中心的"失败原因"字段复用本结构中的 `code`、`message`、`details`，不另起一套异步任务专属错误格式。
