# 一期 14 算法交付跟踪表

## 1. 使用目的

本表用于解决“仓库没有真实 14 算法模型”的交付阻断问题。它把每个算法从 M1 模拟闭环、U6 真实模型准入、M2 算法验收、U7 双硬件联调拆成可跟踪项。

重要边界：

- 模拟算法只能用于 M1 平台闭环，不得计入 M2。
- U6 必须逐算法补齐真实模型、冻结验收集、离线指标、回放测试和人工抽检记录。
- U7 必须在 U6 完成后，再补齐 NVIDIA 与已锁定国产卡的双硬件一致性报告。
- `docs/30-integrations/视觉算法计划支持显卡清单20260617.md` 当前作为硬件候选清单；未明确选定具体国产卡型号、驱动、SDK、镜像和测试机前，不视为国产硬件已锁定。

## 2. 状态定义

| 状态 | 含义 |
| --- | --- |
| `PENDING` | 未提供或尚未开始 |
| `IN_PROGRESS` | 已开始准备，但证据未闭合 |
| `READY_FOR_TEST` | 可进入测试，等待执行报告 |
| `PASS` | 已有可追溯证据并通过 |
| `FAIL` | 已测试但不满足验收 |
| `BLOCKED` | 受外部输入或待确认项阻断 |

## 3. 必填交付证据

| 证据项 | 最低要求 | 验收单元 |
| --- | --- | --- |
| 模型文件 | 模型仓库地址或本地制品路径，包含版本号与 checksum | U6 |
| 模型格式 | ONNX、TensorRT、PyTorch、OM、MagicMind 或厂商格式之一 | U3/U6 |
| 输入输出说明 | 输入尺寸、预处理、后处理、label schema、output schema | U3/U6 |
| 阈值策略 | 默认阈值、租户级覆盖、连续帧策略、人工复核区间 | U6 |
| 冻结验收集 | `FROZEN` 数据集版本、checksum、样本统计、标签分布 | U6 |
| 离线评估报告 | mAP、召回率、精确率等指标达到对应目标 | U6 |
| 视频回放记录 | 回放样本、算法版本、误报漏报记录、结论 | U6 |
| 人工抽检记录 | 抽检人、抽检比例、问题清单、整改结论 | U6 |
| NVIDIA 运行记录 | 后端、驱动、CUDA/TensorRT、镜像 digest、性能数据 | U3/U7 |
| 国产卡运行记录 | 已锁定国产卡型号、驱动、SDK、镜像 digest、性能数据 | U3/U7 |
| 双硬件一致性报告 | 同一模型、同一帧、NVIDIA 与国产后端输出差异不超过阈值 | U3/U7 |

## 7. 推荐推进顺序

1. 先冻结验收集：按 14 算法分别统计样本量、正负样本、场景覆盖和 checksum。
2. 先接入 3 个代表性算法打样：一个检测、一个分割、一个场景/密度类算法。
3. 固化推理制品格式：优先统一 ONNX，再分别转换 NVIDIA TensorRT 与国产后端制品。
4. 每个算法先跑离线指标，达标后再做视频回放。
5. 回放通过后组织人工抽检，抽检问题回流到标注或训练。
6. 国产卡最终型号锁定后，再补跑双硬件一致性和性能基线。
7. 14 个算法全部完成后，才能把 U6 从 `BLOCKED` 改为 `PASS`。

## 8. 当前结论

截至 2026-06-17，仓库已具备 U6 验收规则与跟踪表，但 14 个真实算法交付证据仍未闭合。当前 U6/M2 状态保持 `BLOCKED`，阻断原因从“缺少真实模型”细化为“逐算法缺少模型、冻结集、指标、回放、抽检和双硬件证据”。
