# 部署环境目标与容器编排选型

## 1. 结论

本项目正式开发与交付按两套环境推进：

| 环境 | 目标 | 编排建议 |
| --- | --- | --- |
| 公有云环境 | 客户演示、版本迭代、培训、回归验证 | Docker Compose |
| 江西省交通内网政务云生产环境 | 正式生产运行、真实数据、真实设备、生产验收 | Kubernetes 优先 |

总体原则：

```text
演示环境要轻，生产环境要稳。
Docker Compose 负责快速演示和迭代，Kubernetes 负责生产级治理。
```

## 2. 公有云演示环境

### 2.1 环境定位

公有云环境主要用于：

- 客户阶段性演示。
- 软件版本快速迭代。
- 功能评审。
- 培训与试用。
- M1 模拟闭环展示。
- 回归验证。

该环境可以使用模拟 DJI 网关、模拟推理后端、演示数据和候选硬件池说明，但不得声明为生产真实接入环境。

### 2.2 编排建议

公有云演示环境建议使用 Docker Compose。

原因：

1. Docker Compose 适合用一个 YAML 文件定义并运行多容器应用。
2. 可将前端、后端 API、PostgreSQL/PostGIS、RabbitMQ、Redis、MinIO、DJI 模拟网关、模拟推理服务一键拉起。
3. 部署和重置简单，适合频繁演示、快速改版和演示数据初始化。
4. 运维成本低，不要求客户或项目团队一开始具备 Kubernetes 运维能力。
5. 当前阶段仍存在较多模拟能力和待确认项，Compose 更适合快速闭环。

参考：

- Docker Compose 官方文档：https://docs.docker.com/compose/

### 2.3 公有云环境边界

公有云环境不得承诺：

- 真实 DJI 飞控生产接入。
- 真实 14 算法全部上线验收。
- 国产 AI 加速卡真实硬件验收通过。
- 政务云网络边界、防火墙策略和生产安全策略已验证。
- 客户真实生产数据已接入。

公有云环境可以承诺：

- M1 主链路模拟闭环演示。
- 多租户、权限、功能授权、数据授权逻辑演示。
- 飞控任务、媒体、分析、工单、标注、训练、发布的流程演示。
- U0-U7 规则级测试和验收文档展示。
- 14 算法交付状态跟踪。

## 3. 江西省交通内网政务云生产环境

### 3.1 环境定位

江西省交通内网政务云是生产环境，用于：

- 正式业务运行。
- 真实客户租户和真实生产数据。
- 真实 DJI Dock 3 设备与 DJI Cloud API 适配网关。
- 真实对象存储、消息队列、数据库和网络边界。
- 真实视觉算法模型。
- NVIDIA 与国产 AI 加速卡双硬件适配。
- U7 系统级验收与客户 UAT。

### 3.2 编排建议

生产环境优先使用 Kubernetes。

原因：

1. Kubernetes 官方定位是用于自动化部署、扩缩容和管理容器化应用的生产级容器编排系统。
2. 本项目生产环境包含多个长期运行和异步处理组件，不是单一 Web 应用。
3. 不同组件需要不同的资源、健康检查、副本策略、网络策略和升级策略。
4. Kubernetes 更适合生产环境的滚动升级、故障自恢复、配置管理、密钥管理、资源配额、日志监控和服务治理。
5. 政务云生产环境通常更强调审计、安全边界、服务可用性和可运维性。

参考：

- Kubernetes 官方文档：https://kubernetes.io/docs/

### 3.3 生产环境主要组件

生产环境至少涉及：

| 组件 | 说明 |
| --- | --- |
| Web 前端 | 面向平台方与客户租户的后台页面 |
| API 服务 | REST、WebSocket、认证、租户、权限、业务接口 |
| DJI 适配网关 | 位于 DMZ 或允许外联区域，仅该网关访问 DJI Cloud API |
| 媒体处理 worker | 媒体入库、checksum、分片处理 |
| 抽帧 worker | 视频抽帧和 frame_assets 生成 |
| 分析任务 worker | 分析任务调度、推理队列 |
| 推理服务 | gRPC Predict，适配 NVIDIA 与国产后端 |
| 训练 worker | 数据集训练、评估、模型制品处理 |
| 定时任务 | 清理、补偿、重试、状态巡检 |
| WebSocket 服务 | 遥测和任务进度推送 |
| PostgreSQL/PostGIS | 核心业务数据与空间数据 |
| RabbitMQ | 异步任务消息 |
| Redis | 缓存、会话、限流、在线状态辅助 |
| 对象存储 | 媒体、抽帧、数据集、模型制品 |
| 日志监控 | 审计、指标、告警、运行日志 |

## 4. 过渡策略

如果政务云 Kubernetes 资源、运维权限或容器平台尚未准备好，可设置过渡阶段：

| 阶段 | 公有云演示环境 | 政务云生产环境 |
| --- | --- | --- |
| 第 1 阶段 | Docker Compose | 环境调研、网络边界确认 |
| 第 2 阶段 | Docker Compose + 自动构建镜像 | Docker Compose 单机试运行或 Kubernetes 测试命名空间 |
| 第 3 阶段 | 保持演示与版本迭代 | Kubernetes 部署 U0-U5 主链路 |
| 第 4 阶段 | 演示、培训、回归 | Kubernetes 部署 U6/U7，接真实 DJI、对象存储、算法、硬件 |
| 第 5 阶段 | 非生产演示环境 | Kubernetes 生产发布 |

过渡原则：

- Docker Compose 可用于政务云短期试运行，但不建议作为最终生产形态。
- 如果客户明确接受单机部署、系统规模较小且运维边界已书面确认，才可考虑生产 Compose 方案。
- 默认生产方向仍以 Kubernetes 为准。

## 5. 仓库部署目录建议

建议后续仓库维护以下结构：

```text
infra/
  compose/
    docker-compose.dev.yml
    docker-compose.demo.yml
  k8s/
    base/
    overlays/
      staging/
      production/
```

公有云演示环境启动示例：

```powershell
docker compose -f infra/compose/docker-compose.demo.yml up -d
```

政务云生产环境部署示例：

```powershell
kubectl apply -k infra/k8s/overlays/production
```

## 6. 配置与密钥原则

两套环境必须区分配置：

| 类型 | 公有云演示环境 | 政务云生产环境 |
| --- | --- | --- |
| DJI 凭证 | 默认模拟，不使用真实生产凭证 | 使用真实凭证，存入密钥管理 |
| 数据库 | 演示库，可重置 | 生产库，备份与权限分级 |
| 对象存储 | MinIO 或演示桶 | 政务云/客户认可内网对象存储 |
| 消息队列 | 演示 RabbitMQ | 生产 RabbitMQ 或客户指定消息中间件 |
| 统一身份 | 可使用本地账号 | LDAP/CAS/OAuth2/OIDC 或书面豁免 |
| 日志审计 | 演示留痕 | 满足审计保留策略，至少 180 天 |
| 网络出口 | 可访问必要演示资源 | 仅 DJI 适配网关按白名单外联 |

密钥不得写入代码仓库。生产凭证必须通过环境变量、Kubernetes Secret、政务云密钥服务或客户指定方式注入。

## 7. 与项目验收的关系

| 验收项 | 公有云演示环境 | 政务云生产环境 |
| --- | --- | --- |
| M1 模拟闭环 | 可验收 | 可复核 |
| U0-U5 规则级测试 | 可执行 | 可执行 |
| 真实 DJI 飞控 | 不作为生产验收 | 必须在政务云/真实网络边界验收 |
| 真实 14 算法 | 可展示交付状态 | 必须完成 U6 验收 |
| 国产 AI 加速卡 | 候选池说明 | 必须锁定型号并实测 |
| 网络边界 | 只能演示规则 | 必须做防火墙/白名单/阻断测试 |
| U7 系统级验收 | 不作为生产验收 | 必须执行 |

## 8. 最终决策

本项目后续开发和部署遵循以下决策：

1. 公有云演示与版本迭代环境默认使用 Docker Compose。
2. 江西省交通内网政务云生产环境默认以 Kubernetes 为目标形态。
3. 如政务云暂不具备 Kubernetes 条件，可短期采用 Docker Compose 试运行，但需明确过渡性质和生产限制。
4. 两套环境配置、密钥、数据、外部接口和验收口径必须隔离。
5. 任何模拟能力不得在生产环境验收材料中写成真实能力。
