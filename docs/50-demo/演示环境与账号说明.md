# 演示环境与账号说明

## 1. 当前演示口径

当前可提供的是本地 Web Demo 与命令行 MVP 展示，用于客户阶段性沟通、内部评审和版本迭代说明。

可以展示：
- U0-U7 当前代码进度和自动化测试进度。
- M1 模拟闭环：飞控任务、DJI 模拟回调、媒体入库、抽帧、分析、事件、工单。
- 平台方与客户租户的角色权限差异。
- 客户租户默认无训练、模型发布、平台运维、跨租户统计能力。
- 14 算法交付跟踪状态和真实模型验收缺口。
- NVIDIA 与国产硬件双路线适配边界。

不能承诺：
- 真实 DJI Dock 3 已接入生产。
- 真实 14 个算法模型已全部交付并验收。
- 国产卡型号、驱动、SDK、镜像和测试机已最终锁定。
- 生产统一身份、生产数据库、生产对象存储、消息队列和外部系统已接入。

## 2. 本地 Web Demo

在项目根目录执行：

```powershell

```

浏览器访问：

```text

```

所有演示账号密码相同：

```text
Demo@2026
```

| 账号 | 角色 | 演示重点 |
| --- | --- | --- |
| `platform_admin` | 平台管理员 | 总览、训练发布、14 算法、测试报告、平台方能力边界 |
| `customer_flight` | 客户飞控管理员 | 飞控任务、遥测、媒体同步、M1 模拟链路 |
| `customer_viewer` | 客户分析查看员 | 分析事件、告警、工单 |
| `customer_annotator` | 客户标注员 | 标注任务、数据集状态、客户租户样本授权 |

建议演示顺序：

1. 使用 `platform_admin` 登录，展示 U0-U7 总览和测试 `30/30 PASS`。
2. 查看训练发布与 14 算法，说明真实模型、冻结验收集和双硬件证据仍是 M2 阻断项。
3. 使用 `customer_flight` 登录，展示客户侧飞控任务和 M1 模拟闭环。
4. 使用 `customer_viewer` 登录，展示分析事件、人工复核和工单。
5. 使用 `customer_annotator` 登录，展示标注任务和 `FROZEN` 数据集状态。
6. 用客户账号访问训练发布能力，展示 `FEATURE_403` 权限边界。
