# 外部系统接口适配与内置替代策略

## 1. 结论

一期外部系统接口采用：

```text
适配器接口先行 + 内置替代兜底 + 生产接入单独验收
```

核心原则：

- 不等待所有外部接口确定后才开发主链路。
- 开发、演示、M1 阶段允许使用内置替代。
- 所有内置替代必须标注“待确认 / 非生产真实接入”。
- U7 生产验收必须真实对接，或取得客户书面确认内置替代可作为生产方案。

## 2. 外部系统总体策略

| 外部系统 | 一期建议 | 生产口径 |
| --- | --- | --- |
| OA / 空域审批 | 内置审批流 + 人工审批结果录入 + REST/Webhook 适配器预留 | 有接口则对接；无接口需客户书面确认人工流程 |
| 工单平台 | 先做内置 `work_orders` | 若客户已有工单平台，生产需对接或确认内置替代 |
| 统一身份 | 先用本地账号 | 生产接 LDAP/CAS/OAuth2/OIDC，或形成书面豁免 |
| GIS 平台 | 最小资产台账 + 导入模板 + PostGIS | 生产接客户权威 GIS / 资产系统 |
| 短信 / 邮件 / 通知 | 站内通知 + 通知日志 + 适配器预留 | 生产接客户短信网关、邮件服务器、OA 消息或其他渠道 |
| 无接口场景 | 允许内置替代 | 必须标注边界，不得写成生产接口已接入 |

## 3. OA / 空域审批

### 3.1 一期实现

先实现内置审批能力：

- 内置审批流。
- 审批人手工通过 / 驳回。
- 外部审批编号录入。
- 审批附件或备注。
- REST/Webhook 适配器预留。
- 审批动作写审计。

任务状态流转：

```text
DRAFT -> PENDING_APPROVAL -> APPROVED / REJECTED
```

### 3.2 生产接入

生产环境优先对接客户 OA / 空域审批系统。

如客户无接口或暂不开放接口：

- 必须形成客户书面确认。
- 可使用人工录入审批结果。
- 审批编号、审批人、审批时间、附件和备注必须可追溯。

### 3.3 依据

U1 飞控主链路不能被 OA/空域接口阻塞，但飞行审批属于高风险动作，必须可追溯、可审计。

## 4. 工单平台

### 4.1 一期实现

先实现内置工单：

- `work_orders` 表。
- 工单创建。
- 派单。
- 关闭。
- 驳回。
- 误报 / 漏报回流。
- 事件与工单关联。
- 外部工单号字段预留。
- Webhook / REST 适配器预留。

### 4.2 生产接入

生产环境：

- 客户已有工单平台：优先对接。
- 客户无接口或认可内置：保留内置工单，但需形成确认。

### 4.3 依据

U2 告警到工单是 M1 闭环必需，不能等待外部工单平台接口确认后再开发。

## 5. 统一身份

### 5.1 一期实现

先使用本地账号，按 `docs/20-domain-specs/本地账号认证与Token安全策略.md` 执行。

同时预留统一身份适配器：

- LDAP。
- CAS。
- OAuth2。
- OIDC。

### 5.2 生产接入

生产环境按客户要求接入统一身份。

如客户暂未确认统一身份协议：

- 使用本地账号试运行。
- 标注统一身份待确认。
- 最终生产验收前需客户确认统一身份接入方案或书面豁免。

### 5.3 边界

统一身份只解决：

```text
用户是谁
```

不替代系统内部：

- 租户上下文。
- RBAC。
- `user_tenant_roles`。
- `tenant_feature_entitlements`。
- `tenant_data_grants`。
- 审计日志。

## 6. GIS 平台

### 6.1 一期实现

先实现：

- 最小道路 / 桥梁 / 边坡资产台账。
- GeoJSON / CSV / Excel 导入预留。
- REST / 数据库视图适配器预留。
- `source_crs`。
- `source_system`。
- `external_asset_code`。
- PostGIS 空间字段。

### 6.2 生产接入

生产环境对接客户权威 GIS 或资产系统。

需客户确认：

- GIS 数据源。
- 坐标系。
- 资产编码。
- 桩号映射。
- 更新频率。
- 接口方式。

### 6.3 依据

U1 任务需要绑定资产，U2 事件需要定位到资产。完整 GIS 数据不能阻塞 M1，但生产定位必须可追溯。

本部分与 `docs/20-domain-specs/GIS与资产数据接入策略.md` 配套执行。

## 7. 短信 / 邮件 / 通知

### 7.1 一期实现

先实现统一通知中心：

- 站内通知。
- 通知模板。
- 通知日志。
- 发送状态。
- 失败原因。
- 重试次数。
- 短信、邮件、Webhook 适配器预留。

建议通知状态：

```text
CREATED
PENDING
SENT
FAILED
RETRYING
CANCELLED
```

### 7.2 生产接入

生产环境按客户实际条件对接：

- 客户短信网关。
- 内网邮件服务器。
- OA 消息。
- 企业微信 / 钉钉 / 其他内网消息渠道。

### 7.3 依据

通知不是 M1 主链路硬阻断，但审批、告警、工单、训练、发布、运维告警后续都需要通知能力。

## 8. 无接口时的内置替代规则

没有外部接口时，可以先做内置替代。

| 阶段 | 是否允许内置替代 |
| --- | --- |
| 开发 | 允许 |
| 演示 | 允许 |
| M1 | 允许 |
| 试运行 | 可作为临时兜底 |
| 生产 / U7 | 需真实对接或客户书面确认 |

不得写成：

```text
OA 已接入
GIS 已接入
工单平台已接入
统一身份已接入
```

只能写成：

```text
内置审批流已完成，OA 接口待确认
内置工单已完成，外部工单平台待确认
最小资产台账已完成，权威 GIS 待接入
本地账号已完成，统一身份协议待确认
站内通知已完成，短信/邮件网关待确认
```

## 9. 统一适配器结构

正式工程建议为每类外部系统建立适配器层：

```text
interfaces/
  identity/
  approval/
  gis/
  work_order/
  notification/
```

每个适配器至少支持：

| 能力 | 要求 |
| --- | --- |
| 模式 | `mock` / `internal` / `external` |
| 配置化 | 接口地址、认证、超时、重试可配置 |
| 幂等 | 写接口必须支持幂等 |
| 重试 | 可配置重试次数和退避策略 |
| 超时 | 必须有超时控制 |
| 失败原因 | 统一错误结构 |
| 审计 | 成功、失败、越权、人工替代均写审计 |
| 待确认标记 | 未真实接入时必须明确标注 |

## 10. 数据模型建议

### 10.1 集成配置表

建议建立：

```text
integration_configs
```

字段建议：

| 字段 | 说明 |
| --- | --- |
| `id` | 配置 ID |
| `tenant_id` | 租户 ID，可为空表示平台级 |
| `integration_type` | identity / approval / gis / work_order / notification |
| `mode` | mock / internal / external |
| `name` | 配置名称 |
| `config` | JSONB 配置，不存明文密钥 |
| `status` | ACTIVE / INACTIVE / PENDING_CONFIRMATION |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### 10.2 外部引用字段

业务表建议预留：

| 表 | 字段 |
| --- | --- |
| `mission_approvals` | `external_approval_id`, `external_status`, `external_payload` |
| `work_orders` | `external_work_order_id`, `external_status`, `external_payload` |
| `road_assets / bridge_assets / slope_assets` | `external_asset_code`, `source_system`, `source_crs` |
| `users` | `external_identity_id`, `identity_provider` |
| `notifications` | `external_message_id`, `channel`, `send_status` |

## 11. 错误码建议

外部系统适配错误应复用统一错误码，不新增自定义格式。

| 场景 | 错误码 |
| --- | --- |
| 未认证 / 统一身份失败 | `AUTH_401` |
| 权限不足 | `PERM_403` |
| 租户越权 | `TENANT_403` |
| OA / DJI / 外部服务调用失败 | 优先按领域使用，如 `DJI_502`；其他外部适配可归入结构化 `details` |
| GIS 坐标或资产匹配失败 | `GIS_422` |
| 幂等冲突 | `IDEMP_409` |
| 状态不允许 | `STATE_409` |

如后续确需新增外部适配错误码，必须先更新项目错误码清单，不得模块内自造。

## 12. 验收要求

### 12.1 M1 / 演示

可验收：

- 内置审批流。
- 内置工单。
- 本地账号。
- 最小资产台账。
- 站内通知。
- 适配器配置存在。
- 外部接口待确认状态清晰。

不可声称：

- OA 已真实接入。
- 统一身份已真实接入。
- 权威 GIS 已真实接入。
- 外部工单平台已真实接入。
- 短信 / 邮件网关已真实接入。

### 12.2 U7 / 生产

必须补齐：

- 外部系统接口模式。
- 接口地址与认证方式。
- 网络白名单。
- 超时与重试策略。
- 联调记录。
- 失败回退方案。
- 客户确认或书面豁免。

## 13. 与项目规约的关系

本策略依据：

1. 外部生产接口、真实设备、硬件型号或网络边界未锁定时，不得通过生产验收。
2. DJI 能力必须通过大疆机场 3 适配网关。
3. 统一身份、OA/空域审批、GIS、对象存储、工单平台均属待确认项。
4. 模拟器和内置替代只能用于开发、演示和 M1 平台闭环。
5. 生产验收必须真实对接或客户书面确认替代方案。
6. 所有关键写操作和越权行为必须写审计。

## 14. 正式开发执行原则

后续外部系统相关开发按本文件执行：

1. 每类外部系统必须有适配器接口。
2. 适配器支持 `mock`、`internal`、`external` 模式。
3. 内置替代可以先跑通 M1。
4. 内置替代不得写成生产真实接入。
5. 外部调用必须有超时、重试、失败原因和审计。
6. 生产 U7 前必须完成真实接入或客户确认。
7. 外部系统凭证不得明文入库或进入代码仓库。
