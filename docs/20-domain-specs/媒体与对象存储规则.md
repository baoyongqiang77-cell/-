# 媒体与对象存储规则

## 1. 结论

一期正式开发采用：

```text
媒体格式白名单 + 分片上传 + SHA-256 checksum + 租户前缀对象路径
```

核心决策：

| 问题 | 决策 |
| --- | --- |
| 视频 / 图片格式 | 一期使用格式白名单 |
| 单文件大小限制 | 配置化，按媒体、模型、数据集分别限制 |
| 分片大小 | 默认 8 MB，弱网 4 MB，内网稳定环境 16 MB |
| checksum 算法 | 统一 SHA-256，格式为 `sha256:<hex>` |
| 对象路径规则 | 第一段必须是 `tenant_id`；任务媒体固定 `tenant_id/project_id/mission_id/...` |

## 2. 媒体格式白名单

### 2.1 一期支持格式

| 类型 | 一期支持 | 说明 |
| --- | --- | --- |
| 视频 | MP4、MOV | DJI 设备常见格式，便于抽帧和浏览器预览 |
| 图片 | JPG / JPEG、PNG | 巡检图片、抽帧图片、证据图常用 |
| 缩略图 | JPG、WebP | Web 展示优化 |
| 元数据 | JSON | 遥测、帧地理提示、媒体回调原始信息 |

### 2.2 MIME 白名单

一期允许：

```text
video/mp4
video/quicktime
image/jpeg
image/png
image/webp
application/json
```

不在白名单内的文件返回：

```text
MEDIA_415
```

### 2.3 暂不默认支持格式

以下格式不作为一期默认上传格式：

```text
AVI
MKV
TIFF
RAW
```

如客户生产设备确实输出这些格式，需单独登记为待确认项，并补充解码、预览、抽帧、checksum 和性能测试。

## 3. 单文件大小限制

文件大小限制必须配置化，不能硬编码。

| 场景 | 建议默认限制 |
| --- | --- |
| 图片 | <= 50 MB |
| 普通视频 | <= 5 GB |
| 长视频 / 特殊巡检 | 可配置到 20 GB |
| 模型制品 | <= 20 GB，可按环境配置 |
| 数据集导出包 | <= 50 GB，可按环境配置 |

说明：

- 普通视频默认 5 GB 足以覆盖多数演示和常规巡检视频。
- 超过 20 GB 的视频应走专项上传策略和运维确认。
- 模型制品和数据集导出包可能较大，应与媒体限制分开配置。

## 4. 分片上传规则

### 4.1 分片大小

| 场景 | 建议分片大小 |
| --- | --- |
| 默认 | 8 MB |
| 弱网环境 | 4 MB |
| 内网稳定环境 | 16 MB |
| 超大文件 | 16 MB 或 32 MB |

一期默认：

```text
8 MB
```

依据：

- 分片过小会导致请求数量过多，增加元数据和队列压力。
- 分片过大会提高弱网失败重传成本。
- 8 MB 在请求数量和弱网恢复之间较均衡。

### 4.2 分片上传流程

分片上传必须支持：

1. 上传初始化。
2. 分片登记。
3. 分片 checksum。
4. 断点续传。
5. 重复分片幂等。
6. 合并完成。
7. 完整文件 checksum 校验。
8. 失败重试。
9. 过期清理。

### 4.3 分片状态

建议状态：

```text
CREATED
UPLOADING
UPLOADED
VERIFIED
FAILED
EXPIRED
```

重复上传同一分片时：

- 分片 checksum 一致：幂等返回成功。
- 分片 checksum 不一致：返回 `MEDIA_499`。

## 5. checksum 规则

### 5.1 算法

统一使用：

```text
SHA-256
```

格式：

```text
sha256:<hex>
```

示例：

```text
sha256:9f2c...
```

### 5.2 校验层级

| 对象 | 是否必须 |
| --- | --- |
| 分片 checksum | 建议支持 |
| 完整文件 checksum | 必须 |
| 抽帧图片 checksum | 必须 |
| 数据集 checksum | 必须 |
| 模型制品 checksum | 必须 |
| 冻结验收集 checksum | 必须 |
| 评估报告 / 回放记录 checksum | 建议支持 |

checksum 不一致返回：

```text
MEDIA_499
```

### 5.3 依据

统一 SHA-256 的原因：

1. 项目 API 示例已使用 `sha256:...`。
2. SHA-256 通用性强，跨语言、跨服务易实现。
3. 媒体、数据集、模型制品、冻结验收集都需要可追溯校验。
4. 有利于 U2 弱网和断点续传验收。
5. 有利于 U6 冻结验收集和模型制品验收。

## 6. 对象路径规则

### 6.1 核心原则

对象路径第一段必须是：

```text
tenant_id
```

任何媒体、抽帧、数据集、模型制品、报告对象都不得省略租户前缀。

### 6.2 任务相关对象路径

任务媒体、帧、事件截图固定使用：

```text
tenant_id/project_id/mission_id/{category}/{object_id}.{ext}
```

示例：

```text
t_customer_001/proj_001/ms_20260616_0007/media/med_8841.mp4
t_customer_001/proj_001/ms_20260616_0007/chunks/upload_001/000001.part
t_customer_001/proj_001/ms_20260616_0007/frames/frame_000001.jpg
t_customer_001/proj_001/ms_20260616_0007/snapshots/evt_001.jpg
```

### 6.3 非任务对象路径

数据集、模型、报告不一定有 `mission_id`，使用：

```text
tenant_id/datasets/...
tenant_id/models/...
tenant_id/reports/...
```

示例：

```text
t_customer_001/datasets/ds_001/export/ds_001_coco.zip
t_jxjtsz_platform/models/alg_road_crack/mv_001/runtime/nvidia/model.plan
t_jxjtsz_platform/reports/u6/road_crack/evaluation_report.pdf
```

### 6.4 category 建议

| category | 用途 |
| --- | --- |
| `media` | 原始视频、原始图片 |
| `chunks` | 分片临时对象 |
| `frames` | 抽帧图片 |
| `snapshots` | 事件证据截图 |
| `annotations` | 标注导出或标注辅助文件 |
| `datasets` | 数据集导出包 |
| `models` | 模型制品 |
| `reports` | 评估报告、回放报告、UAT 材料 |

## 7. 数据库记录建议

### 7.1 media_files

`media_files` 至少包含：

| 字段 | 说明 |
| --- | --- |
| `tenant_id` | 租户 ID |
| `mission_id` | 飞行任务 ID |
| `source_type` | DJI_DOCK_3 / MANUAL_UPLOAD / EXTERNAL_IMPORT |
| `media_type` | VIDEO / IMAGE / FRAME / THUMBNAIL |
| `mime_type` | MIME 类型 |
| `storage_uri` | 对象存储路径 |
| `checksum` | 完整文件 SHA-256 |
| `size_bytes` | 文件大小 |
| `status` | 媒体状态 |
| `captured_at` | 采集时间 |
| `uploaded_at` | 上传完成时间 |

### 7.2 media_chunks

`media_chunks` 至少包含：

| 字段 | 说明 |
| --- | --- |
| `tenant_id` | 租户 ID |
| `media_file_id` | 媒体文件 ID |
| `upload_id` | 上传会话 ID |
| `chunk_index` | 分片序号 |
| `chunk_size` | 分片大小 |
| `chunk_checksum` | 分片 checksum |
| `storage_uri` | 分片对象路径 |
| `status` | 分片状态 |

建议唯一约束：

```text
upload_id + chunk_index
```

### 7.3 frame_assets

`frame_assets` 至少包含：

| 字段 | 说明 |
| --- | --- |
| `tenant_id` | 租户 ID |
| `media_file_id` | 源媒体文件 |
| `mission_id` | 飞行任务 |
| `frame_index` | 帧序号 |
| `frame_time` | 帧时间 |
| `storage_uri` | 帧图片对象路径 |
| `checksum` | 帧图片 checksum |
| `geo_hint` | 帧地理提示 JSONB |
| `status` | 帧状态 |

## 8. 状态与错误码

媒体状态必须使用项目固定枚举：

```text
CAPTURED
UPLOADING
UPLOAD_FAILED
REGISTERING
READY
REJECTED
FRAME_EXTRACTING
FRAME_READY
ARCHIVED
DELETED
```

错误码：

| 错误码 | 触发场景 |
| --- | --- |
| `MEDIA_415` | 媒体格式或 MIME 不支持 |
| `MEDIA_499` | 分片上传未完成、checksum 不一致、文件不完整 |
| `TENANT_403` | 对象路径租户前缀与当前租户不一致 |
| `TENANT_404` | 当前租户上下文下资源不存在 |
| `IDEMP_409` | 幂等键冲突 |

## 9. 对象存储适配

开发和演示环境：

- 使用 MinIO。
- 使用 S3 兼容 API。
- 可重置 bucket 和演示数据。

生产环境：

- 使用客户认可的政务云或内网对象存储。
- 优先要求 S3 兼容。
- 如非 S3 兼容，必须通过对象存储适配器封装。

业务代码不得直接依赖某一厂商 SDK，应通过统一接口：

```text
ObjectStorageClient
```

至少提供：

- `init_multipart_upload`
- `upload_part`
- `complete_multipart_upload`
- `abort_multipart_upload`
- `head_object`
- `get_object`
- `put_object`
- `delete_object`
- `generate_presigned_url`

## 10. 验收要求

### 10.1 U2 必测项

| 测试项 | 通过标准 |
| --- | --- |
| 格式白名单 | 不支持格式返回 `MEDIA_415` |
| 对象路径租户隔离 | 路径第一段必须为当前 `tenant_id` |
| 分片断点续传 | 弱网中断后可继续上传 |
| 重复分片 | checksum 一致时幂等成功 |
| checksum 不一致 | 返回 `MEDIA_499` |
| 重复回调 | 不重复入库 |
| 抽帧产物 | frame_assets 可追溯到 media_files 和 mission |
| 跨租户访问 | 返回 `TENANT_403` 或 `TENANT_404` |

### 10.2 U6 / U7 必测项

| 测试项 | 通过标准 |
| --- | --- |
| 冻结验收集 checksum | 数据集冻结后 checksum 不可变 |
| 模型制品 checksum | runtime_artifacts 记录 checksum |
| 报告对象路径 | 报告路径包含租户或平台方前缀 |
| 生产对象存储 | 客户认可对象存储读写、校验、权限通过 |

## 11. 与项目规约的关系

本规则依据：

1. 项目要求媒体、抽帧、数据集、模型制品必须对象路径租户隔离。
2. API 示例已采用 `tenant_id/project_id/mission_id/media/...` 路径。
3. U2 验收重点包括弱网、断点续传、重复回调、checksum。
4. U6 冻结验收集和模型制品必须有 checksum。
5. 客户租户不得推断或访问其他租户对象。
6. 生产对象存储接口仍是待确认项，因此必须通过适配器而不是写死具体厂商。

## 12. 正式开发执行原则

后续 U2 媒体入库、抽帧、对象存储和 U6/U7 制品管理按本文件执行：

1. 格式白名单先行。
2. 不支持格式返回 `MEDIA_415`。
3. 分片默认 8 MB，可配置。
4. checksum 统一 SHA-256。
5. 完整文件 checksum 必须校验。
6. 对象路径第一段强制 `tenant_id`。
7. 任务媒体固定 `tenant_id/project_id/mission_id/...`。
8. 数据集、模型、报告使用 `tenant_id/datasets`、`tenant_id/models`、`tenant_id/reports`。
9. 开发环境用 MinIO，生产通过对象存储适配器接客户认可存储。
10. 模拟对象存储不得写成生产对象存储验收通过。
