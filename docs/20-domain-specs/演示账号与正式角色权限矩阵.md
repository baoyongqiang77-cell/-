# 演示账号与正式角色权限矩阵

## 1. 结论

本项目正式账号与角色设计遵循：

```text
平台方角色与客户租户角色分离
最小权限
租户隔离
功能授权
数据授权
关键操作审计
```

正式系统中，权限不得直接绑定用户，应通过：

```text
users -> user_tenant_roles -> roles -> permissions / feature entitlements
```

一个用户可以在不同租户拥有不同角色，也可以在同一租户叠加多个角色，但高风险权限必须显式授权并写审计。

## 2. 正式角色矩阵

| 角色 | 归属 | 默认功能 | 不应默认拥有 |
| --- | --- | --- | --- |
| 平台管理员 | 平台方 | 租户管理、用户角色、功能授权、系统配置、审计查看 | 不直接默认执行模型发布，可叠加算法工程师角色 |
| 平台运维员 | 平台方 | 设备接入、DJI 网关状态、任务监控、日志告警、受控跨租户运维 | 模型发布、训练审批、数据集导出 |
| 算法工程师 | 平台方 | 数据集、训练任务、模型评估、模型版本、灰度发布、回滚 | 租户管理、平台运维、客户账号管理 |
| 客户飞控管理员 | 客户租户 | 本租户设备、航线、飞行任务、审批、遥测、媒体同步 | 跨租户设备、训练、模型发布、平台运维 |
| 客户分析查看员 | 客户租户 | 本租户分析任务、事件、告警、工单、证据查看 | 飞控下发、标注管理、训练、发布 |
| 客户标注员 | 客户租户 | 本租户授权标注任务、样本查看、标注提交 | 数据集导出、训练、模型发布、其他租户样本 |
| 审计查看员 | 平台方 / 客户租户 | 平台方可按授权看全局或跨租户审计；客户只能看本租户审计 | 写操作、审批、发布、导出 |

## 3. feature_code 映射建议

| 角色 | FLIGHT_CONTROL | VISION_ANALYSIS_RESULT | DATA_ANNOTATION | DATASET_EXPORT | MODEL_TRAINING | MODEL_RELEASE | CROSS_TENANT_STATS | PLATFORM_OPS |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 平台管理员 | 是 | 是 | 是 | 是 | 可选 | 可选 | 是 | 是 |
| 平台运维员 | 是 | 是 | 否 | 否 | 否 | 否 | 可选 | 是 |
| 算法工程师 | 否 | 是 | 是 | 是 | 是 | 是 | 可选 | 否 |
| 客户飞控管理员 | 是 | 可选 | 否 | 否 | 否 | 否 | 否 | 否 |
| 客户分析查看员 | 否 | 是 | 否 | 否 | 否 | 否 | 否 | 否 |
| 客户标注员 | 否 | 可选 | 是 | 默认否 | 否 | 否 | 否 | 否 |
| 审计查看员 | 否 | 否 | 否 | 否 | 否 | 否 | 按归属 | 平台方可选 |

说明：

- `可选` 表示可通过角色叠加或专项授权开通，但不作为该角色默认权限。
- 客户租户即使具备某个功能授权，也只能访问本租户资源。
- 客户租户默认不开通 `DATASET_EXPORT`、`MODEL_TRAINING`、`MODEL_RELEASE`、`CROSS_TENANT_STATS`、`PLATFORM_OPS`。
- 客户数据进入平台方训练、评估或跨租户统计前，必须存在有效 `tenant_data_grants`。

## 5. 平台方角色说明

### 5.1 平台管理员

默认职责：

- 租户创建、停用、查看。
- 用户、角色、租户范围授权。
- 功能授权配置。
- 系统配置与字典维护。
- 审计日志查看。

限制：

- 不默认执行模型发布。
- 不默认直接处理训练任务。
- 如需训练和发布能力，应叠加算法工程师角色或走专项授权。

### 5.2 平台运维员

默认职责：

- 设备接入。
- DJI 网关运行状态。
- 任务监控。
- 日志告警。
- 受控跨租户运维。
- 网络边界和服务健康检查。

限制：

- 不默认拥有模型发布。
- 不默认拥有训练审批。
- 不默认拥有数据集导出。

### 5.3 算法工程师

默认职责：

- 数据集管理。
- 训练任务。
- 模型评估。
- 模型版本管理。
- runtime_artifacts 管理。
- 灰度发布。
- 回滚。

限制：

- 不默认管理租户。
- 不默认管理客户账号。
- 不默认执行平台运维。

## 6. 客户租户角色说明

### 6.1 客户飞控管理员

默认职责：

- 本租户设备查看和管理。
- 本租户航线管理。
- 本租户飞行任务创建。
- 本租户任务审批。
- 本租户遥测查看。
- 本租户媒体同步查看。

限制：

- 不能访问其他租户设备、航线、任务和遥测。
- 不能训练模型。
- 不能发布模型。
- 不能访问平台运维。

### 6.2 客户分析查看员

默认职责：

- 查看本租户分析任务。
- 查看本租户检测结果。
- 查看本租户事件、告警、工单。
- 查看本租户证据图和复核结果。

限制：

- 不默认创建飞行任务。
- 不默认处理标注任务。
- 不默认训练和发布模型。
- 不允许跨租户查看结果。

### 6.3 客户标注员

默认职责：

- 查看本租户授权标注任务。
- 查看被分配样本。
- 提交标注。
- 查看标注退回和复检结果。

限制：

- 不默认导出数据集。
- 不默认训练模型。
- 不默认发布模型。
- 不得处理其他租户样本。

### 6.4 审计查看员

平台方审计查看员：

- 可按授权查看平台方审计。
- 可按受控租户切换查看跨租户审计。

客户租户审计查看员：

- 只能查看本租户审计。

限制：

- 不具备写操作。
- 不具备审批。
- 不具备发布。
- 不具备导出。

## 7. 权限控制依据

本角色矩阵依据项目规约中的以下原则：

1. 系统必须采用“平台方租户 + 客户租户”的逻辑多租户模式。
2. 平台方租户 `tenant_type` 为 `PLATFORM_OPERATOR`。
3. 客户租户 `tenant_type` 为 `CUSTOMER_TENANT`。
4. 客户租户默认只开放本租户范围内飞控、数据标注、视觉分析结果、告警和工单能力。
5. 客户租户默认关闭训练任务、模型发布、跨租户统计、平台运维和数据集导出能力。
6. 未授权功能统一返回 `FEATURE_403`。
7. 客户数据进入平台方训练、评估或跨租户统计前，必须存在有效 `tenant_data_grants`，否则返回 `DATA_GRANT_412`。
8. 写操作、审批、发布、删除、越权、跨租户切换必须写审计。

## 8. 审计要求

以下操作必须写 `audit_logs`：

| 操作 | 审计要求 |
| --- | --- |
| 登录 / 登出 | 记录账号、租户、IP、时间 |
| 创建 / 修改 / 删除租户 | 记录操作者、前后状态 |
| 角色授权 / 回收 | 记录被授权用户、角色、租户 |
| 功能授权配置 | 记录 feature_code、租户、前后状态 |
| 数据授权配置 | 记录 owner_tenant、grantee_tenant、用途、范围、有效期 |
| 飞行任务审批 / 下发 | 记录任务状态变化和操作者 |
| 训练任务创建 | 记录数据授权校验结果 |
| 模型发布 / 回滚 | 记录模型版本、目标租户、状态变化 |
| 数据集导出 | 记录导出范围、文件路径、操作者 |
| 越权访问拒绝 | 记录错误码、资源类型、租户上下文 |
| 跨租户切换 | 记录原租户、目标租户、操作者和原因 |

## 9. 正式开发执行原则

后续 U0 认证/RBAC 初始化按本文件执行：

1. 先初始化平台方租户和至少两个客户租户。
2. 初始化上述正式角色。
3. 初始化演示账号，但仅用于演示环境。
4. 通过 `user_tenant_roles` 绑定用户、租户、角色。
5. 通过 `tenant_feature_entitlements` 控制租户功能授权。
6. 通过 `tenant_data_grants` 控制客户数据进入训练、评估或跨租户统计。
7. 客户租户访问未授权训练、发布、导出、平台运维时返回 `FEATURE_403`。
8. 未授权客户数据进入训练时返回 `DATA_GRANT_412`。
9. 所有越权拒绝和关键写操作写审计。
